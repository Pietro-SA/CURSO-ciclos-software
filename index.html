El ciclo de vida del software y sus modelos

Para lograr llevar a cabo el desarrollo de un software de forma efectiva y beneficiosa, es necesario saber gestionar el TIEMPO, los RECURSOS, y además cumplir con sus REQUERIMIENTOS.

¿Como se describe el processo de desarrollo?

El ciclo de vida de un software tiene un início, medio y final. Se inicia con la necesidad de crear un programa que solucione un problema o cumpla una función, entonces este recibe mantenimiento para corregir errores e implementar nuevas funciones, y cuando el programa ya no es de utilidad, sea porque está defasado o dejó de ser utilizado, llega su final y ya no recibe actualizaciones.

El modelo en cascada

Este processo de desarrollo está organizado en etapas que se ejecutan en un orden predeterminado. Es decir, de manera inflexible. Es muy útil cuando ya se tienen los objectivos preestablecidos y hay buena comunicación entre el cliente y desarrollador. Puede llegar a ser problemático si se necesita entregar el software antes de que el modelo de desarrollo ya se haya finalizado, o no se ha hecho un buen planeamiento para el proyecto.
